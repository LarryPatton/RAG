│  ⬜ 搜索中       │                              │

**系统状态：**
- 向量索引已加载（首次启动约 7 秒，后续从缓存加载约 5 秒）
- 大模型已就绪（Ollama 本地或云端 API）
- 等待用户输入

---

### Step 1：用户输入需求

**用户操作：** 在对话框中输入自然语言需求

**典型输入示例：**
- "帮我找一款 500 以内的降噪耳机，主要通勤用"
- "推荐个游戏耳机，头戴式的"
- "有没有运动时戴的蓝牙耳机，200 左右"
- "我想买个耳机"（信息不足，系统会追问）

**系统行为：**
1. 将用户消息发送给 AI Agent
2. Agent 分析用户意图，提取已知约束条件
3. Agent 判断信息是否充足，决定追问还是直接搜索

---

### Step 2：意图澄清（多轮对话）

**用户看到的：**

```
👤 帮我找一款500以内的降噪耳机，主要通勤用

🤖 [意图澄清] 好的！500元以内的降噪耳机，通勤场景，
   我来帮你挑选。还想确认几个细节：

   1. 耳机类型：你更喜欢入耳式（小巧便携）还是头戴式
     （包耳舒适）？
   2. 有没有特别偏好的品牌？比如 Sony、华为、小米等？
```

**侧边栏状态：**
```
🔄 意图澄清  ← 当前
⬜ 搜索中
⬜ 推荐方案
⬜ 订单确认
⬜ 下单完成
```

**推理日志新增：**
```
[10:23:05] 用户输入
  输入: 帮我找一款500以内的降噪耳机，主要通勤用
[10:23:08] 意图澄清
  输出: 提取约束 — 预算≤500，降噪，通勤；追问类型和品牌
```

**系统行为：**
- Agent 识别出：预算 ≤ 500、需要降噪、场景 = 通勤
- Agent 判断缺失信息：类型偏好、品牌偏好
- Agent 生成追问，而非直接搜索

**分支逻辑：**
- 如果用户第一句话信息非常少（如"我想买个耳机"），Agent 会追问更多问题
- 如果用户给了充分信息（如"帮我找 Sony 的入耳降噪耳机 500 以内通勤用"），Agent 仍会确认至少一个细节（如"除了 Sony 还考虑其他品牌吗？"）

---

### Step 3：用户补充信息

**用户操作：** 回答 Agent 的追问

**用户看到的：**

```
👤 入耳式就好，国产品牌优先
```

**系统行为：**
- Agent 更新约束条件：类型 = 入耳式，品牌偏好 = 国产
- Agent 判断信息充足，开始搜索商品库

> 如果信息仍不充足，Agent 会继续追问（最多 2-3 轮）。

---

### Step 4：商品检索

**用户看到的：**

```
🤖 [搜索中] 好的，为你搜索500元以内、入耳式、国产品牌
   降噪耳机，适合通勤使用...
```

（此时显示加载动画 "思考中..."）

**侧边栏状态：**
```
✅ 意图澄清
🔄 搜索中  ← 当前
⬜ 推荐方案
⬜ 订单确认
⬜ 下单完成
```

**系统行为：**
1. Agent 调用 `product_search` 工具
2. 工具将查询转为语义向量
3. 在 500 条商品的向量索引中检索 Top-8 匹配
4. 返回 8 条商品的完整信息（名称、价格、品牌、评分、特点）
5. Agent 根据用户约束（预算、类型、品牌偏好）从 8 条中筛选出 3 条最佳推荐

**推理日志新增：**
```
[10:23:15] 搜索中
  输入: 500以内入耳式国产降噪耳机通勤
  输出: 检索到8条结果，top1: 小米 Buds 4 Pro ¥599...
```

---

### Step 5：推荐方案展示

**用户看到的：**

```
🤖 [推荐方案] 根据你的需求，为你精选了3款国产入耳式
   降噪耳机：

   | 商品名称          | 价格 | 评分 | 核心特点              | 推荐理由           |
   |-------------------|------|------|-----------------------|-------------------|
   | 小米 Redmi Buds 5 Pro | ¥299 | 4.4 | 52dB降噪、LDAC      | 降噪最强，性价比王 |
   | 华为 FreeBuds 5i     | ¥449 | 4.5 | 主动降噪、LDAC、多设备| 音质均衡，生态好   |
   | 漫步者 NeoBuds Pro 2 | ¥499 | 4.6 | 50dB降噪、Hi-Res     | 音质最好，降噪强   |

   💡 个人推荐：小米 Redmi Buds 5 Pro。
   虽然价格最低，但 52dB 降噪深度在这个价位无敌，
   LDAC 高音质加持，地铁通勤场景表现优秀。

   你觉得哪款更合适？或者有其他要求？
```

**侧边栏状态：**
```
✅ 意图澄清
✅ 搜索中
🔄 推荐方案  ← 当前
⬜ 订单确认
⬜ 下单完成
```

**分支逻辑 — 用户可能的回应：**

| 用户回应 | 系统处理 |
|---------|---------|
| "第一款不错" | → 进入 Step 6（订单确认） |
| "有没有更便宜的？200以内" | → Agent 更新预算，重新搜索（回到 Step 4） |
| "头戴式的有吗？" | → Agent 更新类型，重新搜索（回到 Step 4） |
| "第二款和第三款再详细对比一下" | → Agent 展开对比分析，留在当前步骤 |
| "这几个都不喜欢，有没有 JBL 的" | → Agent 更新品牌偏好，重新搜索（回到 Step 4） |

## 特殊场景

### 场景 A：用户中途改需求

```
👤 帮我找降噪耳机，500以内
🤖 [意图澄清] 入耳式还是头戴式？
👤 入耳式
🤖 [推荐方案] 推荐3款...
👤 预算放宽到1000，有什么更好的？
🤖 [搜索中] 好的，为你重新搜索1000元以内的...
🤖 [推荐方案] 新的3款推荐...
```

系统会自动感知需求变化，重新检索并推荐。

### 场景 B：用户需求模糊

```
👤 我想买个耳机
🤖 [意图澄清] 好的！帮你挑耳机。先了解几个问题：
   1. 预算大概多少？
   2. 主要什么场景用？通勤、运动、办公还是游戏？
   3. 有没有类型偏好？入耳式还是头戴式？
👤 通勤用，300左右
🤖 [意图澄清] 收到，需要降噪功能吗？通勤降噪体验会好很多
👤 要降噪
🤖 [搜索中] 搜索中...
```

Agent 会通过多轮追问逐步明确需求。

### 场景 C：用户直接选择不下单

```
👤 帮我找游戏耳机
🤖 [意图澄清] ...
👤 头戴式，500以内
🤖 [推荐方案] 推荐3款...
👤 谢谢，我先了解一下
🤖 好的，有需要随时找我！
```

用户无需下单即可结束对话。

## 侧边栏功能说明

| 功能 | 说明 |
|------|------|
| LLM 选择 | 切换本地模型 / 云端 API，切换后对话清空 |
| 清空对话 | 重置所有对话记录和推理日志 |
| 阶段进度条 | 5 个阶段的实时进度（⬜ → 🔄 → ✅） |
| 推理日志 | 每一步的时间戳、输入、输出，可展开查看详情 |

侧边栏的推理日志是演示亮点 — 让观众实时看到 AI 在"想什么、做什么"。
