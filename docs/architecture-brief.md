# RAG 智能购物助手 — 技术架构说明

> 面向产品经理的简洁版，不涉及代码细节。

---

## 一句话概括

用户用自然语言描述需求 → AI 助手多轮对话理解意图 → 从商品库中智能检索 → 跨平台比价 + 库存查询 → 推荐对比 → 一键下单。

## 核心能力

### 1. 多轮对话理解需求

不是关键词搜索，是真正理解自然语言：

| 用户说的 | 系统理解的 |
|---------|----------|
| "帮我找一款500以内的降噪耳机，通勤用" | 预算≤500、需要降噪、使用场景=通勤 |
| "入耳式，国产品牌优先" | 补充条件：入耳式、品牌偏好=国产 |
| "预算再低一点，300以内" | 更新预算≤300，其他条件保持 |

AI 在信息不足时会提供快捷回复按钮（如"入耳式""头戴式"），用户点击即可快速补充信息，无需手动输入。只需"预算+类型"或"预算+场景"即可开始搜索。

前端同时通过正则实时提取用户输入中的决策信息（类型、预算、场景、降噪、品牌），同步展示在右侧决策面板。

### 2. 信息门控机制

Agent 不会在信息不足时盲目搜索。Gate 节点会在每次工具调用前检查：

- 扫描全部对话历史，提取已确认的用户偏好
- 5 个决策维度：预算、耳机类型、使用场景、降噪需求、品牌偏好
- **至少确认 3 个维度**才放行工具调用
- 不足时自动注入反馈，引导 LLM 继续追问用户
- 最多 3 次重试防止无限循环

### 3. 语义检索（RAG）

传统搜索："降噪耳机" → 只匹配包含这四个字的商品

语义检索："通勤地铁上用的，不想听到噪音" → 理解意图，匹配所有降噪耳机

检索流程：
1. 使用 BGE-small-zh-v1.5 向量模型将查询转为向量
2. 从 Qdrant（内存模式）中检索 Top-30 相似商品
3. 按类型严格过滤（入耳式/头戴式/骨传导/耳挂式）
4. 按预算硬过滤（不返回超预算商品）
5. 返回 Top-8 给 LLM，由 AI 筛选出 3 条推荐

首次构建向量索引约 90 秒，之后通过 `index_cache/embeddings.npz` 缓存，约 2 秒加载。

### 4. 跨平台比价 + 库存查询

AI 自动为候选商品在京东、天猫、拼多多三大平台进行比价，同时查询实时库存，确保推荐的商品有货且价格最优。

### 5. 结构化推荐 + 下单

AI 以卡片形式展示 3 款商品对比（品牌、价格、评分、优劣势、推荐理由），右侧决策面板同步展示候选漏斗和比价表格。用户点击选择 → 弹出订单确认 → 确认下单。

后端缓存推荐商品数据（LRU，最多 100 条），在订单确认阶段用缓存价格覆盖 LLM 输出，防止价格幻觉。

### 6. 实时流式交互

所有 AI 回复通过 SSE 逐字流式输出，工具调用过程实时展示（开始/结束），用户全程可见 AI 的"思考过程"。

前端实现 30 秒超时检测 + 自动重试（最多 2 次），保障流式连接稳定性。

## 技术选型

| 层级 | 技术 | 版本 | 作用 | 为什么选它 |
|------|------|------|------|-----------|
| 大模型 | Qwen 2.5 (14B) / Qwen-Plus | — | 理解意图、生成回复 | 中文能力强，支持本地+云端双模式 |
| Agent 框架 | LangGraph | ≥ 0.2 | 编排 AI 决策流程（状态机） | 支持条件路由、信息门控、流式事件 |
| LLM 抽象 | LangChain | ≥ 0.3 | 统一 LLM 接口 | 生态丰富，Ollama/通义千问无缝切换 |
| 检索框架 | LlamaIndex | ≥ 0.11 | 语义搜索商品 | 轻量、支持自定义嵌入和后处理 |
| 向量数据库 | Qdrant | ≥ 1.12 | 存储和检索向量 | 内存模式适合 Demo，支持过滤 |
| 向量模型 | BGE-small-zh-v1.5 | — | 将文本转为向量 | 中文专用、免费、本地运行 (~100MB) |
| 后端 | FastAPI | ≥ 0.115 | API 网关、SSE 流式传输 | 异步高性能，原生支持 SSE |
| 前端 | React + Vite + Tailwind | 19 / 8 / 4 | 聊天界面 + 决策面板 | 组件化开发，热更新，响应式布局 |
| 图标 | Lucide React | 0.469 | UI 图标 | 轻量、一致性好 |
| Markdown | react-markdown | 10.1 | 渲染 AI 回复中的 Markdown | 支持 GFM，可定制渲染 |

## 后端数据流

```
用户消息 + 历史对话
        │
        ▼
┌─ Agent Service (单例) ───────────────────────────────┐
│                                                       │
│  1. 正则提取用户决策（预算/类型/场景/降噪/品牌）        │
│  2. 构建决策前缀（注入已确认信息到用户消息）             │
│  3. 调用 LangGraph Agent (astream_events v2)          │
│  4. 流式分发事件：                                     │
│     · token → 逐字输出                                │
│     · tool_start/end → 工具调用状态                    │
│  5. 解析累积文本中的 JSON 块                           │
│     · recommendation → 商品推荐                       │
│     · order_confirm → 订单确认                        │
│     · task_plan → 任务计划                            │
│     · quick_replies → 快捷回复                        │
│  6. 缓存推荐商品（LRU, max 100）                      │
│  7. 订单确认价格用缓存覆盖（防幻觉）                    │
│  8. 分发 structured_data / decision_update / stage     │
│                                                       │
└───────────────────────────────────────────────────────┘
```

## SSE 流式事件

前后端通过 Server-Sent Events 进行实时通信，事件类型包括：

| 事件类型 | 用途 |
|---------|------|
| `token` | LLM 输出的文本片段，逐字追加 |
| `tool_start` | 工具开始调用（搜索、比价、库存等），含工具名和图标 |
| `tool_end` | 工具调用完成，返回结果 |
| `task_plan` | 任务计划 JSON，前端展示步骤预览 |
| `structured_data` | 推荐方案 / 订单确认的结构化 JSON |
| `decision_update` | 用户决策更新（从推荐中提取 user_profile） |
| `stage` | 当前工作流阶段变更（只进不退） |
| `thinking_steps` | 工具调用汇总 |
| `quick_replies` | 快捷回复按钮选项 |
| `done` | 流结束标记，附带清理后的纯文本 |

## 可扩展方向

- **接入真实电商 API**：替换模拟工具为京东/淘宝接口
- **增加商品品类**：目前仅耳机，可扩展至全品类
- **接入更多工具**：优惠券查询、物流追踪、价格监控
- **多模态**：支持图片搜索（"类似这款的耳机"）
- **用户画像**：记住用户偏好，个性化推荐
- **持久化存储**：当前 Qdrant 使用内存模式，可切换为持久化部署
- **对话历史持久化**：支持多会话管理和历史回溯
